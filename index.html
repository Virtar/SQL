Cuestionario de Ejercicios SQL

Fundamentos de Consultas SQL

Los siguientes ejercicios te ayudarán a practicar con las cláusulas básicas de SQL, incluyendo SELECT, DISTINCT, ORDER BY, WHERE, filtros condicionales, entre otros.

Ejercicio 1: Consulta store_id, first_name y last_name de la tabla customer de la base de datos sakila.

Utiliza la cláusula SELECT para seleccionar las columnas específicas de la tabla.

Estructura: SELECT columna1, columna2, columna3 FROM esquema.tabla;

Ejercicio 2: Cambia el nombre de las columnas store_id, first_name y last_name a Tienda, Nombre y Apellido respectivamente.

Puedes usar alias para cambiar los nombres de las columnas en el resultado.

Estructura: SELECT columna1 AS NuevoNombre1, columna2 AS NuevoNombre2 FROM esquema.tabla;

También puedes omitir la palabra AS: SELECT columna1 NuevoNombre1, columna2 NuevoNombre2 FROM esquema.tabla;

Ejercicio 3: Ordena de manera descendente la columna Apellido en la consulta del ejercicio 2.

Utiliza la cláusula ORDER BY con DESC para ordenar de manera descendente.

Estructura: SELECT columna1 Alias1, columna2 Alias2 FROM esquema.tabla ORDER BY Alias2 DESC;

Nota: Puedes usar el alias en el ORDER BY.

Ejercicio 4: ¿Cuál es la cantidad más baja y más alta de la columna amount en la tabla payment?

Puedes usar las funciones MIN() y MAX() para encontrar los valores mínimo y máximo.

Estructura: SELECT MIN(columna), MAX(columna) FROM esquema.tabla;

Alternativa: También podrías usar ORDER BY con LIMIT 1 para cada caso.

Ejercicio 5: Traer los distintos precios y ordenarlos de manera Ascendente.

Utiliza DISTINCT para eliminar duplicados y ORDER BY para ordenar.

Estructura: SELECT DISTINCT columna FROM esquema.tabla ORDER BY columna ASC;

Nota: ASC es el orden predeterminado, por lo que es opcional incluirlo.

Ejercicio 6: Traer solo los precios distintos de la tabla payment donde el amount sea menor a 3 y ordenarlos de manera descendente.

Combina DISTINCT, WHERE y ORDER BY.

Estructura: SELECT DISTINCT columna FROM esquema.tabla WHERE columna < valor ORDER BY columna DESC;

Ejercicio 7: Traer todos los datos del staff donde el staff_id sea distinto de 2.

Utiliza el operador != o <> para "distinto de".

Estructura: SELECT * FROM esquema.tabla WHERE columna != valor;

Ejercicio 8: Traer todos los idiomas que sean distintos de German.

Utiliza WHERE con el operador de desigualdad.

Estructura: SELECT * FROM esquema.tabla WHERE name != "German";

Ten en cuenta que el valor de texto debe estar entre comillas.

Ejercicio 9: Consulta description, release_year de la tabla film donde title sea IMPACT ALADDIN.

Usa SELECT para las columnas específicas y WHERE para filtrar por título.

Estructura: SELECT columna1, columna2 FROM esquema.tabla WHERE title = "IMPACT ALADDIN";

Nota: Las consultas SQL generalmente distinguen entre mayúsculas y minúsculas para los valores de texto.

Ejercicio 10: Consulta la tabla payment donde amount sea mayor a 0.99 y ordenarlo de manera descendente.

Combina WHERE para filtrar y ORDER BY para ordenar.

Estructura: SELECT * FROM esquema.tabla WHERE precio > 0.99 ORDER BY precio DESC;

Ejercicio 11: Traer todos los registros en la tabla country donde el country sea igual a Algeria y además el country_id = 2.

Utiliza el operador AND para combinar condiciones.

Estructura: SELECT * FROM esquema.tabla WHERE condicion1 AND condicion2;

Ambas condiciones deben cumplirse para que se devuelva el registro.

Ejercicio 12: Traer todos los registros en la tabla country donde el country sea igual a Algeria o el country_id = 12.

Utiliza el operador OR para combinar condiciones de forma que se cumpla al menos una.

Estructura: SELECT * FROM esquema.tabla WHERE condicion1 OR condicion2;

Basta con que una condición sea verdadera para que se devuelva el registro.

Ejercicio 13: Traer todos los registros en la tabla language donde el language_id sea igual a 1 o el name = german.

Similar al ejercicio anterior, usa OR para unir las condiciones.

Estructura: SELECT * FROM esquema.tabla WHERE language_id = 1 OR name = "german";

Ejercicio 14: Traer todos los registros en la tabla category donde no se encuentre el name Action.

Utiliza el operador != o NOT para negar una condición.

Estructura: SELECT * FROM esquema.tabla WHERE name != "Action";

Alternativa: SELECT * FROM esquema.tabla WHERE NOT name = "Action";

Ejercicio 15: Traer todos los registros en la tabla film donde el rating no sea PG.

Similar al ejercicio anterior, utiliza != para excluir un valor específico.

Estructura: SELECT * FROM esquema.tabla WHERE rating != "PG";

Ejercicio 16: Traer todos los distintos tipos de rating y que no esté contemplado el 'PG'.

Combina DISTINCT con una condición de exclusión usando !=.

Estructura: SELECT DISTINCT rating FROM esquema.tabla WHERE rating != "PG";

Ejercicio 17: Filtra la información en la tabla payment donde customer_id sea igual a 36, amount sea mayor a 0.99 y staff_id sea igual a 1. Ordenarlos de manera ascendente por amount.

Utiliza múltiples condiciones con AND y añade ORDER BY.

Estructura: SELECT * FROM esquema.tabla WHERE customer_id = 36 AND amount > 0.99 AND staff_id = 1 ORDER BY amount ASC;

Ejercicio 18: Traer todos los nombres donde sean Mary y Patricia de la tabla customer.

Puedes usar el operador IN como alternativa a múltiples condiciones OR.

Estructura: SELECT * FROM esquema.tabla WHERE first_name IN ("Mary", "Patricia");

Equivalente a: SELECT * FROM esquema.tabla WHERE first_name = "Mary" OR first_name = "Patricia";

Ejercicio 19: Traer todos los registros de la tabla film donde las características especiales sean Trailer y Trailer,Deleted Scenes, además el rating tiene que ser G y R y el tiempo de duración tiene que ser mayor a 50. Ordenarlos de manera ascendente por tiempo.

Combina IN con AND y una comparación numérica.

Estructura: SELECT * FROM esquema.tabla WHERE special_features IN (...) AND rating IN (...) AND length > 50 ORDER BY length ASC;

Usa IN para los valores categóricos y comparadores para los valores numéricos.

Ejercicio 20: Traer todos los registros menos Action, Children, Animation de la tabla category.

Utiliza NOT IN para excluir múltiples valores.

Estructura: SELECT * FROM esquema.tabla WHERE name NOT IN ("Action", "Children", "Animation");

Ejercicio 21: Consulta la tabla film_text de la base de datos sakila. Filtra la información donde title sea ZORRO ARK, VIRGIN DAISY, UNITED PILOT.

Utiliza el operador IN para filtrar por múltiples títulos.

Estructura: SELECT * FROM esquema.tabla WHERE title IN ("ZORRO ARK", "VIRGIN DAISY", "UNITED PILOT");

Ejercicio 22: Consulta la tabla city de la base de datos sakila. Filtra la información donde city sea Chiayi, Dongying, Fukuyama y Kilis.

Similar al ejercicio anterior, usa IN para incluir una lista de ciudades.

Estructura: SELECT * FROM esquema.tabla WHERE city IN ("Chiayi", "Dongying", "Fukuyama", "Kilis");

Ejercicio 23: Traer todas las rentas donde el customer_id sean del 300 al 500, y además sean del staff_id = 1. Ordenarlo de manera ASC por customer_id.

Utiliza BETWEEN para el rango de IDs y AND para la condición adicional.

Estructura: SELECT * FROM esquema.tabla WHERE customer_id BETWEEN 300 AND 500 AND staff_id = 1 ORDER BY customer_id ASC;

BETWEEN incluye los valores límite (300 y 500 en este caso).

Ejercicio 24: Traer todos los registros de la tabla payment donde el monto sea entre 3 y 5, ordenar por amount de manera DESC.

Utiliza BETWEEN para el rango de montos y ORDER BY para ordenar.

Estructura: SELECT * FROM esquema.tabla WHERE amount BETWEEN 3 AND 5 ORDER BY amount DESC;

Ejercicio 25: Traer todos los registros de la tabla payment donde el monto no sea entre 3 y 5, ordenar por amount de manera DESC.

Utiliza NOT BETWEEN para excluir un rango de valores.

Estructura: SELECT * FROM esquema.tabla WHERE amount NOT BETWEEN 3 AND 5 ORDER BY amount DESC;

Alternativa: SELECT * FROM esquema.tabla WHERE amount < 3 OR amount > 5 ORDER BY amount DESC;

Ejercicio 26: Consulta la tabla payment de la base de datos sakila. Filtra la información donde amount esté entre 2.99 y 4.99, staff_id sea igual a 2 y customer_id sea 1 y 2.

Combina BETWEEN, AND e IN para las diferentes condiciones.

Estructura: SELECT * FROM esquema.tabla WHERE amount BETWEEN 2.99 AND 4.99 AND staff_id = 2 AND customer_id IN (1, 2);

Ejercicio 27: Consulta la tabla address de la base de datos sakila. Filtra la información donde city_id esté entre 300 y 350.

Utiliza BETWEEN para el rango de city_id.

Estructura: SELECT * FROM esquema.tabla WHERE city_id BETWEEN 300 AND 350;

Ejercicio 28: Consulta la tabla film de la base de datos sakila. Filtra la información donde rental_rate esté entre 0.99 y 2.99, length sea menor igual de 50 y replacement_cost sea menor de 20.

Combina BETWEEN y operadores de comparación (<=, <) con AND.

Estructura: SELECT * FROM esquema.tablam WHERE rental_rate BETWEEN 0.99 AND 2.99 AND length <= 50 AND replacement_cost < 20;

Operador LIKE

El operador LIKE se utiliza en la cláusula WHERE para buscar un patrón específico en una columna de texto. Usa caracteres comodín como % (coincide con cualquier número de caracteres) y _ (coincide con un único carácter).

Ejercicio 29: Traer todos los registros en la tabla actor, los nombres que empiecen con la letra "A" y el apellido empiece con la letra "B".

Utiliza el operador LIKE con el comodín % al final del patrón.

El símbolo % representa cero o más caracteres.

Estructura: SELECT * FROM esquema.tabla WHERE first_name LIKE 'A%' AND last_name LIKE 'B%';

Esto buscará nombres que empiecen con "A" y apellidos que empiecen con "B".

Ejercicio 30: Traer todos los registros donde los nombres terminen con la letra "A" y además los apellidos terminen con la letra "N".

Utiliza el operador LIKE con el comodín % al inicio del patrón.

Estructura: SELECT * FROM esquema.tabla WHERE first_name LIKE '%A' AND last_name LIKE '%N';

El símbolo % al principio indica cualquier número de caracteres antes de la letra final.

Ejercicio 31: Traer todos los registros donde el nombre contiene la palabra "NE" en cualquiera de sus posiciones y además que tenga el apellido la palabra "RO".

Utiliza el operador LIKE con el comodín % tanto al inicio como al final del patrón.

Estructura: SELECT * FROM esquema.tabla WHERE first_name LIKE '%NE%' AND last_name LIKE '%RO%';

El patrón '%NE%' buscará "NE" en cualquier posición del nombre (inicio, medio o final).

Ejercicio 32: Traer todos los registros donde todos los nombres empiecen con "A" y terminen con "E".

Utiliza el operador LIKE con el comodín % en el medio del patrón.

Estructura: SELECT * FROM esquema.tabla WHERE first_name LIKE 'A%E';

Esto buscará nombres que empiecen con "A", tengan cualquier número de caracteres en medio, y terminen con "E".

Ejercicio 33: Traer todos los registros donde los nombres empiecen con la letra "C" y terminen con "N" y además que el apellido empiece con la letra "g".

Combina patrones LIKE para diferentes columnas.

Estructura: SELECT * FROM esquema.tabla WHERE first_name LIKE 'C%N' AND last_name LIKE 'g%';

Ejercicio 34: Consulta la tabla film de la base de datos sakila. Filtra la información donde release_year sea igual a 2006 y title empiece con ALI.

Combina patrones LIKE para diferentes columnas.

Estructura: SELECT * FROM esquema.tabla WHERE condicion1 = valor AND condicion2 LIKE 'ALI%';

Consejos para resolver estos ejercicios

1. Lee atentamente cada ejercicio para entender qué se te pide exactamente.

2. Identifica las cláusulas y operadores SQL necesarios (SELECT, WHERE, ORDER BY, etc.).

3. Ten en cuenta las tablas y esquemas que debes consultar (sakila.tabla).

4. Recuerda que los textos deben ir entre comillas ("texto") en las condiciones.

5. Verifica que tu consulta final cumpla con todos los requisitos del ejercicio.
